RAG/语音面试官项目对齐说明（详细版）

一、你想做什么（目标与体验）
- 产品形态：AI 语音面试官，面试官主动提问
- 岗位切换：不同岗位进入不同面试官（AI/算法/HR/网络安全/数据库/数据分析/测试等）
- 语言：中文为主
- 交互方式：语音对话（STT -> LLM -> TTS）
- 面向大众部署，云 API 可用
- 输出：面试报告（评分 + 改进建议）
- 数据要求：所有题库/资料都要被用到
- 结构要求：组件独立、结构清晰、互不影响，不因为单点故障崩溃

二、你描述的面试官体验（完整细化）
- 进入面试：用户选择岗位/方向（例如 AI、网络安全）
- 主动提问：面试官从题库中选题，并根据回答做追问
- 题目来源：题库为主 + 适度生成题混合（后续可调配比例）
- 追问策略：根据回答中的缺失点与薄弱点追问
- 评分逻辑：对每题评分，并按维度汇总
  - 技术深度
  - 表达清晰度
  - 结构化思维
  - 关键点覆盖度
  - 实践落地能力（如岗位需要）
- 面试报告：面试结束后输出
  - 总分 + 维度得分
  - 优势与短板
  - 关键遗漏点
  - 改进建议与学习路径
  - 可选：推荐下一轮练习题目

三、已对齐的技术方向
1) 语音链路
- 当前选择：STT -> LLM -> TTS（语音转文字 -> 大模型 -> 文字转语音）
- 选择理由：可控、可记录、便于评分与报告
- 未来可扩展端到端语音，但仍保留文字记录

2) RAG/知识库
- 全量入库，但允许分批执行
- 精准度优先（不追求速度优先）
- 本地优先验证效果（有 GPU），后续可迁移到云/数据库
- OCR 可接受（用于扫描版/图片型文件）
